机器学习 Machine Learning 周志华

第 1 章 绪论

1.1 引言

  • 何为机器学习？

研究如何通过计算的手段，利用系统经验来改善系统自身的性能。

研究的主要内容是关于计算机上从数据中产生”模型“的算法，即”学习算法“

更形式化的一个定义，用P来评估计算机程序在某任务类T上的性能，若一个程序通过利用经验E在T中任务上获得了性能改善，则我们就说关于T和P，该程序对E进行了学习。

1.2 基本术语

  • 数据集 data set 一组记录的集合
  • 示例 instance 或 样本 sample 每条记录是关于一个事件或对象的描述
  • 属性或特征 反映事件或对象在某方面的表现或性质的事项
  • 属性值
  • 属性空间 样本空间 或 “输入空间”
  • 维数
  • 学习/训练: 从数据中学得模型的过程
  • 训练数据：训练过程中用的数据
  • 训练样本：训练过程中使用的每个样本
  • 训练集：训练样本组成的集合
  • 假设 hypothesis：学得模型对应数据的某种潜在规律
  • 真实 ground-truth：潜在规律自身
  • 分类 classification：预测离散值的学习任务
  • 回归 regression：预测连续值的学习任务
  • 二分类 binary classification：正类 positive class 和 反类 negative class
  • 多分类任务 multi-class classification：涉及多个类别
  • 测试 testing：学得模型后使用模型对其进行预测
  • 测试样本：被预测的样本
  • 聚类 clustering：将训练集中的训练样本分成若干组，每组成为一个簇 cluster
  • 监督学习 supervised learning：分类和回归是它的代表
  • 无监督学习：训练数据无标记信息，聚类是它的代表
  • 泛化能力 generalization：学得模型适用于新样本的能力

1.3 假设空间

  • 归纳 induction：从特殊到一般的泛化过程
  • 演绎 deduction：从一般到特殊的特化过程
  • 归纳学习 inductive learning：从样例中学习
  • 版本空间 version space：存在着一个与训练集一致的“假设集合”

1.4 归纳偏好

  • 归纳偏好 inductive bias：机器学习算法在学习过程中对某种类型假设的偏好
  • 奥卡姆剃刀原则
  • NFL定理 No Free Lunch Theorem

1.5 发展历程

  • 20世纪八十年代，符号主义学习，其中包括决策树和基于逻辑的学习
  • 20世纪九十年代中期以前，另一主流技术是基于神经网络的连接主义学习
  • 20世纪九十年代中期，统计学习，支持向量机Support Vector Machine SVM和kernel methods
  • 21世纪初，深度学习，卷积神经网络

第 2 章 模型评估与选择

2.1 经验误差与过拟合

  • 错误率：分类错误的样本数占样本总数的比例
  • 精度：1-错误率
  • 误差：学习器的实际预测输出与样本的真实输出之间的差异
  • 训练误差/经验误差：学习器在训练集上的误差
  • 泛化误差：在新样本上的误差
  • 过拟合 overfitting：学习器将训练样本的一些特点当做了所有潜在样本都有的一般性质
  • 欠拟合 underfitting：对训练样本的一般性质都没学好
  • 模型选择问题

2.2 评估方法

通过实验测试来对学习器的泛化误差进行评估并进而做出选择

使用测试集来测试学习器对新样本的判别能力，以测试集上的测试误差作为泛化误差的近似。

通过对包含m个样例的数据集D进行适当的处理产生训练集S和测试集T

2.2.1 留出法

将数据集D划分为两个互斥的集合，一个作为训练集S，一个作为测试集T

训练/测试集的划分需要保持数据分布的一致性，避免数据划分带来的偏差

常见做法以2/3~4/5的样本用于训练，剩余样本用于测试

2.2.2 交叉验证法

将D划分为k个分层采样得到的大小近似的集合Di，将其中的某个Di定为测试集，其他的均为训练集，可以得到k次训练和测试，返回k个测试结果的均值，又称为“k折交叉验证”

k最常用的取值是10

由于将数据集D划分为k个子集存在多种划分方式，为减少因样本划分不同的差别，可以随机使用不同的划分重复p次，称为p次k折交叉验证，最常见的有10次10折交叉验证

留一法 Leave-One-Out LOO：当D包含m个样本时，令k=m，则需要进行m次训练，比较准确但是计算开销过大

2.2.3 自助法

自助法 bootstrapping

以自助采样为基础，从含m个样本的D中取出1个样本再将其放回，取m次，就得到了含有m个样本的数据集D’，某个样本在m次采样中始终不会被采到的概率为e 即0.368

以D’作训练集，D\D’为测试集，这样仍有数据总量约1/3的没在D’中出现的样本用于测试，结果称为包外估计 out-of-bag estimate

自助法在数据集较小难以划分测试集和训练集时较有用，但是会改变原本数据集的分布，带来估计偏差，故在数据集较大的情况下还是常用留出法和交叉验证法

2.2.4 调参与最终模型

调参 parameter tuning

当进行模型评估和学习中，不仅要对适用学习算法进行选择，还需要设定算法参数

为了区分，模型评估和选择中用于评估测试的数据集称为“验证集”

在研究对比不同算法的泛化性能时，我们用测试集上的判别效果来估计模型实际使用的泛化能力，将训练数据分为训练集和验证集，基于验证集上的性能进行模型选择和调参

2.3.2 查准率、查全率与F1

以信息检索为例子，查准率代表“检索出的信息中用户感兴趣的信息比例”，如邮件系统的垃圾箱和商品推荐，查全率代表“用户感兴趣的信息有多少被检索出来了”，如逃犯信息检索或灾害预测经常用到

查准率和查全率是矛盾的，一高一低。参考在一堆西瓜中挑好瓜。

P-R图，以查全率为横轴，以查准率为纵轴，按从正例可能性从大到小对每个样本作为正例来预测

书上的PR图仅作示意效果，有一点不太准确，如何理解这个图呢？

第一次预测时，预测最有可能的样本为正例，其余m-1个样本均预测为反例，此时大概率有P=1/1，但是也有可能出现最有可能的样本被误判的情况，所以并不是一定从1起始。R=1/1+FN，接近是从0开始产生0

第k次预测时，预测k个样本为正例，m-k个反例，P=TP/k，R=TP/(TP+FN)，很明显当k=m时，FN=0，R=1，P=TP/m并不一定为0

PR图上的点越往右上角靠，性能越好

如果拿python自己跑一下这个图能理解得更直观深刻

查准率、查全率的性能度量有两种方法

  • BEP度量：比较两者相等时的取值

  • F1度量： $$F1=\frac{2TP}{样例总数+TP-TN}$$
